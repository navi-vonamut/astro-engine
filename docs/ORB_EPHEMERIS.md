# Орбы и Эфемериды — формулы и пояснения

Здесь собраны ключевые формулы и объяснения, используемые в `KerykeionEngine`.

1) Абсолютное положение и знак
- `abs_pos` — абсолютный градус от 0 до 360: 0° = 0° Овна.
- `sign_id = floor(abs_pos / 30)` (0..11)
- `degree = abs_pos % 30` (0..30)

2) Орб (минимальная круговая разность)

$$
\Delta = |\theta_1 - \theta_2| \\
\text{orb} = \min(\Delta,\; 360 - \Delta)
$$

3) Определение состояния аспекта (алгоритм `_get_aspect_state`)
- Пусть `t_pos` — абсолютная позиция транзитной планеты, `t_speed` — её скорость (deg/day).
- Пусть `n_pos` — позиция натальной планеты, `aspect_angle` — целевой угол (0,60,90,120,180).
- Вычисляем текущую орбу: `current_orb = |min(|t_pos-n_pos|, 360-|t_pos-n_pos|) - aspect_angle|`.
- Сдвигаем транзитную планету на небольшой шаг `dt = 0.1` дня: `next_pos = (t_pos + t_speed * dt) % 360`.
- Вычисляем `next_orb` аналогично.
- Если `current_orb < 0.1` → `exact`.
- Иначе, если `next_orb < current_orb` → `applying` (или `retrograde_applying`, если `t_speed < 0`).
- Иначе → `separating` (или `retrograde_separating`).

Комментарий: выбор `dt = 0.1` дня даёт небольшую проекцию движения на ~2.4 часа, достаточную для оценки тренда.

4) Ретроградность и стационарность
- Скорость берётся через swisseph (`swe.calc_ut`) при доступе к PID объекта.
- `is_retro = speed < 0`
- `is_stationary = abs(speed) < 0.05` (deg/day) — порог в коде.

5) Комбустность
- Планета считается сожжённой (combust), если её орб к Солнцу < 8.5°:

$$
\text{combust} = (\text{orb}(planet, Sun) < 8.5) 
$$

6) Построение эфемериды (алгоритм в `graphical_ephemeris`)
- Для каждой даты от `start_date` до `end_date` (включительно) с шагом `step_days`:
  - Создаётся `BirthInput` на 12:00 локального времени.
  - Строится `subject = build_subject(t_inp)` (kerykeion).
  - Для списка целевых точек (константа `TARGET_POINTS_BASE` + узлы) извлекаются `abs_pos`.
  - Сохраняется запись `{ date: YYYY-MM-DD, <PointName>: abs_pos, ... }`.
- Возвращается массив таких записей.

7) Точность и замечания
- `kerykeion` даёт удобные объекты и распределение домов; swisseph используется как резерв для точных скоростей и фолбек-позиций.
- Для критических вычислений (микросекунды/минуты) следует учитывать, что `transits` строятся на 12:00 локального времени — это упрощение и может не подходить для задач, требующих минутной точности.

8) Пороговые значения и константы (в коде)
- `STATIONARY_THRESHOLD = 0.05` deg/day — порог стационарности
- `COMBUST_ORB = 8.5` degrees
- `EXACT_ORB_THRESHOLD` (используется в `_get_aspect_state`) ≈ 0.1 degrees
