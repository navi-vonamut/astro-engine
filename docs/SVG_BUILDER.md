# Документация: `svg_builder` (render)

Файл реализации: `app/engine/render/svg_builder.py`

Коротко
- Функция `build_natal_svg(subject) -> str` генерирует SVG-изображение натальной карты на основе объекта `subject` (объект `AstrologicalSubject` из `kerykeion`) и возвращает SVG как строку.

Зависимости и предпосылки
- Требуется установленный пакет `kerykeion` и корректно настроенные его ресурсы (шрифты, шаблоны), т.к. `KerykeionChartSVG` использует их при генерации SVG.
- Функция использует временную директорию (`tempfile.TemporaryDirectory`) — артефакты удаляются после чтения.

Сигнатура
- Вход: `subject` — объект, получаемый, например, через `AstrologicalSubjectFactory.from_birth_data(...)`.
- Выход: `str` — текст SVG (UTF-8). Пример: строка, начинающаяся с `<?xml` или `<svg`.

Поведение
- Создаётся временная директория.
- Инициализируется `KerykeionChartSVG(subject, chart_type="Natal", new_output_directory=tmpdir)`.
- Вызывается `chart.makeSVG()` — в результате в `tmpdir` генерируется файл `*.svg`.
- Скрипт читает первый найденный `.svg` файл и возвращает его содержимое как строку.

Особенности и рекомендации
- Генерация SVG — относительно тяжёлая операция (использует swisseph/kerykeion), поэтому:
  - Вызывать только по необходимости (например, по кнопке на фронтенде).
  - Кешировать результат, если один и тот же SVG генерируется часто (например, по `subject` hash).
- Ошибки: текущая реализация не перехватывает исключения; рекомендуется оборачивать вызов на уровне роутера и возвращать корректный HTTP 500 с логированием.
- Альтернативный способ отдачи SVG напрямую клиенту (без JSON):

```python
from fastapi import Response
svg = build_natal_svg(subject)
return Response(content=svg, media_type="image/svg+xml")
```

- Если вы отдаёте SVG внутри JSON (как сейчас в `/natal_svg`), учтите: фронтенд должен корректно вставлять HTML/SVG (например, через `innerHTML` или `dangerouslySetInnerHTML` в React). Следует валидировать/экранировать, если SVG может содержать небезопасные данные.

Пример использования (в коде сервера)

```python
from app.engine.render.svg_builder import build_natal_svg
# subject — объект AstrologicalSubject
svg_text = build_natal_svg(subject)
# вернуть как JSON
return {"status": "success", "svg": svg_text}
```

Пример использования (фронтенд)

- Вставка в DOM (опасно, но удобно):

```js
// React example
<div dangerouslySetInnerHTML={{ __html: svgString }} />
```

- Или сохранить в файл и отдать через `<img src="/static/generated.svg" />`.

Потенциальные проблемы
- Если `KerykeionChartSVG` использует внешние шрифты или ресурсы, убедитесь, что контейнер/сервер имеет доступ к этим ресурсам.
- Память/время: генерация тяжелая — при пиковых нагрузках возможны таймауты; подумайте об offload'е (очередь задач) для большого количества запросов SVG.

Что можно улучшить
- Добавить опциональные параметры: размер изображения, темы/цвета, включение/исключение слоёв (аспекты/дома и т.д.).
- Добавить обработку ошибок внутри `build_natal_svg` и логирование.
- Вернуть метаинформацию (время генерации, используемый шаблон) вместе с SVG, например: `{ svg: "...", meta: {...} }`.
