# Документация: `app/engine/core`

Этот документ описывает модели, утилиты и константы, которые составляют «ядро» движка.

Файлы:
- `app/engine/core/models.py` — `BirthInput` (входная модель для всех расчётов)
- `app/engine/core/utils.py` — вспомогательные функции для дат/времён, JD, домов и времени
- `app/engine/core/constants.py` — маппинг swisseph, правила аспектов, списки знаков и т.д.

---

## models.py

Класс:
- `BirthInput` — dataclass(frozen=True)
  - Параметры: `name: str, date: str, time: str, tz: str, lat: float, lon: float, house_system: str = "P", node_type: str = "true"`
  - Использование: создаётся в роутерах (см. `app/routes/*.py`) и передаётся в методы движка (`KerykeionEngine`, `GeoAstroEngine`).

Примечание: `frozen=True` делает объект неизменяемым, что удобно для кэширования и безопасной передачи между функциями.

## utils.py

Главные функции и их поведение:

- `norm_date(s: str) -> str`
  - Приводит дату к виду `YYYY-MM-DD` (замена `/` на `-`, триминг).

- `tz_to_pytz(tz: str) -> str`
  - Конвертирует короткий формат `+03:00` в строку для `pytz` (`Etc/GMT-3`).
  - Если передан IANA-таймзон (`Europe/Warsaw`) — возвращает её.
  - В случае ошибки возвращает `UTC`.

- `parse_ymd(date_str: str) -> tuple[int,int,int]`
  - Разбивает `YYYY-MM-DD` на `(year, month, day)`.

- `parse_hm(time_str: str) -> tuple[int,int]`
  - Разбивает `HH:MM[:SS]` на `(hour, minute)`; при ошибке выбрасывает `ValueError`.

- `get_house_for_degree(degree: float, houses: list[dict]) -> int`
  - Находит номер дома (1..12) для абсолютного градуса `degree` по списку `houses`, где каждый дом имеет поле `abs_pos` (начальная абсолютная долгота дома) и `house`.
  - Учитывает цикличность (c1 > c2 — переход через 360°).

- `measure_time(func)`
  - Декоратор для измерения времени выполнения функции (по умолчанию результат не печатается, но строка форматирования оставлена для дебага).

- `get_utc_jd_from_input(inp) -> float`
  - Универсальный конвертер `BirthInput` в Юлианский день UTC (swisseph `swe.julday`), учитывает `tz` через `pytz` локализацию.
  - Парсит дату/время, учитывает секунды, пытается корректно применить смещение по часовому поясу; при ошибках использует UTC и логирует предупреждение.

Рекомендации:
- `tz_to_pytz` простая и предполагает, что смещение записано без минут (MM должно быть 00); для надежного парсинга временных зон (DST и т.п.) используйте IANA-зоны.
- `get_utc_jd_from_input` использует `pytz.localize` — убедитесь, что значение `tz` корректно задано (например, `Europe/Moscow` или результат `tz_to_pytz`).

## constants.py

Ключевые константы:

- `SWISSEPH_OBJECTS` — словарь: имя точки -> swisseph PID. Содержит классические планеты, Хирон, Mean Lilith, узел и распространённые астероиды (Ceres, Pallas, Juno, Vesta).

- `ASPECT_RULES` — правила аспектов: ключ — угол, значение — кортеж `(name, orb_for_planets, orb_for_fictives)`.
  - Примеры: `0: ("conjunction", 8.0, 2.5)`, `60: ("sextile", 6.0, 2.0)`, `180: ("opposition", 8.0, 2.5)` и т.д.

- `STRICT_POINTS` — список точек (узлы, фикивные точки, углы), для которых используется строгий орб.

- `SIGNS_FULL`, `SIGNS_SHORT` — именование знаков (полные и короткие).

- `KERYKEION_HOUSES` — имена атрибутов в объекте `subject` Kerykeion для домов (`first_house`, `second_house`, ...).

- `SIGN_RULERS_BY_ID` — карта: индекс знака (0..11) -> правящая планета (современные управители для некоторых знаков).

- `TARGET_POINTS_BASE` — список точек, которые извлекаются при построении натала (классические + Chiron + Mean Lilith + астероиды).

---

Где используются
- Константы импортируются в `kerykeion_engine.py` для парсинга точек и генерации выходного JSON.
- `ASPECT_RULES` используется в `aspects_calc.py` для определения орбисов и типов аспектов.
- `SWISSEPH_OBJECTS` используется по всему движку для вызовов `swe.calc_ut`.
