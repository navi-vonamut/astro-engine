# Документация: `app/geo` данные и `cities.py`

Этот документ описывает, откуда берутся данные городов, формат JSON и как использовать утилиту `get_major_cities()` в коде.

Файлы
- `app/geo/cities.py` — загрузчик и кешер списка городов.
- `app/geo/data/major_cities.json` — основной источник данных (список крупных городов).

Функция `get_major_cities()`
- Подпись: `get_major_cities() -> List[Dict]`
- Поведение: лениво загружает `major_cities.json` при первом вызове и кеширует результат в памяти (`_CITIES_CACHE`).
- На ошибку чтения или при отсутствии файла возвращает пустой список и печатает предупреждение/ошибку в логах.

Формат `major_cities.json` (каждый элемент массива):
- `name` (string) — англ. имя города, например "Moscow".
- `lat` (number) — широта в десятичных градусах.
- `lon` (number) — долгота в десятичных градусах.
- `country` (string) — ISO-2 код страны (например, "RU").
- `population` (int) — население (опционально, но присутствует в файле).
- `timezone` (string) — IANA зона, например `Europe/Moscow`.

Пример записи:
{
  "name": "Moscow",
  "lat": 55.75222,
  "lon": 37.61556,
  "country": "RU",
  "population": 10381222,
  "timezone": "Europe/Moscow"
}

Рекомендации по использованию
- Если вы вызываете `get_major_cities()` часто в течение жизни процесса, кэширование устраняет повторное чтение файла.
- Для больших задач (фильтрация, географическая выборка) лучше предварительно преобразовать список городов в numpy-массивы или spatial-индекс (R-tree) для ускорения поиска.
- Поля `population` и `timezone` могут использоваться для ранжирования и локализации.

Обновление базы городов
- В репозитории `major_cities.json` уже содержит ~топ городов.
- Если нужно обновить или заменить набор, просто перезапишите `app/geo/data/major_cities.json` — при следующем рестарте сервиса файл загрузится.
- Если вы обновляете файл на горячую (без перезапуска), нужно очистить `_CITIES_CACHE` вручную (в `cities.py`) или вызвать процесс перезагрузки.

Лицензия и источник данных
- В JSON-файле нет встроенных метаданных о лицензии данных. Рекомендуется добавить README или комментарий с указанием источника (GeoNames, Natural Earth, MaxMind и т.д.) и условий использования.

Пример использования

```python
from app.geo.cities import get_major_cities

cities = get_major_cities()
# Пример: выбрать города с населением > 5M
big_cities = [c for c in cities if c.get('population', 0) > 5_000_000]
```

Интеграция с `GeoAstroEngine`
- `GeoAstroEngine.calculate_city_scores_combined` ожидает список городов в том же формате (каждый элемент должен содержать `lat`, `lon`, `name` хотя бы).
- Функция возвращает список городов с добавленным полем `aspects` и `is_crossing`.

Ошибки и диагностика
- Если вы видите `WARNING: Файл городов не найден` — проверьте путь `app/geo/data/major_cities.json` и права доступа.
- Ошибки парсинга выводятся в лог: `ERROR: Ошибка чтения базы городов: <msg>`.

Дополнения (опционально)
- Можно добавить CLI-скрипт `scripts/generate_cities.py` (в репозитории есть `app/geo/scripts/generate_cities.py`) для регенерации `major_cities.json` из внешнего источника.
- Рекомендуется добавить небольшой `meta` объект в JSON (в начале файла) с указанием `source`, `date_generated`, `license`.
